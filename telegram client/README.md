# Telegram Bot для системы тестирования

## Описание
Бот является клиентским модулем распределённой платформы для опросов и тестирования.  
Он принимает команды от пользователей и взаимодействует с главным API.  
Авторизация и хранение токенов реализованы через Redis.

## Текущие команды
- `/login` — авторизация (сохранение токена в Redis)
- `/logout` — выход из системы (удаление токена)
- `/check` — проверка токена

## Будущие команды
- `/users` — список пользователей
- `/user {id}` — информация о пользователе
- `/courses` — список курсов
- `/course {id}` — данные о курсе
- `/addcourse {json}` — создать курс
- `/test {id}` — получить тест
- `/question {id}` — получить вопрос
- `/answer {test_id} {user_id}` — ответы пользователя

## Технологии
- Go
- Telegram Bot API
- Redis
- REST API (главный модуль)

План развития клиента (Telegram Bot):

1. Базовый каркас:
   - Авторизация через /login
   - Выход через /logout
   - Проверка токена через /check

2. Интеграция с API:
   - Команды для пользователей (/users, /user {id})
   - Команды для курсов (/courses, /course {id}, /addcourse)
   - Команды для тестов (/test {id}, /addtest, /deltest)
   - Команды для вопросов (/question {id}, /addquestion)

3. Улучшения:
   - Обработка ошибок (401, 403)
   - Логирование действий
   - Подсказки по командам (/help)

4. Дальнейшее развитие:
   - Поддержка inline-кнопок
   - Поддержка многошаговых диалогов
   - Масштабирование через Docker/Kubernetes
